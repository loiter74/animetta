Typography

Animetta is a small-text product. Most copy lives at 12–14px and the page feels denser than a typical web app. There is no custom webfont — we lean on each OS's native CJK font so Chinese characters render at their proper hinting weight.

The font stack

font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Hiragino Sans", "Noto Sans SC", "Microsoft YaHei", sans-serif;

macOS users get San Francisco + Hiragino. Windows users get Segoe + YaHei. Linux falls back to Noto Sans SC. Each OS therefore renders Animetta in the font its CJK rasterizer is tuned for — there are no missing glyph rectangles on Windows and no chunky pseudo-bold faux-rendering on macOS.

Weight & line-height

We carry only three weights. Body sits on a generous 1.5 leading because CJK glyphs need vertical room — anything tighter feels squashed.

Line-heights

1.15 tight
Display copy & the Animetta wordmark. Compresses pairs of big letters so the brand feels confident.
1.30 snug
Chat bubbles + subtitle overlay. Just loose enough to read fluently, just tight enough that two-line bubbles still look like one thought.
1.50 body
Default. Settings descriptions, all multi-line system copy. Gives CJK glyphs the vertical air they need.
1.65 relax
Long-form helper text — onboarding paragraphs, persona descriptions, release notes.

Working rules

① No third weight per block.

Pick one weight per UI block. Mixing 500 and 600 inside the same bubble or button label flattens the hierarchy.

② Numerals always mono.

Timestamps, latencies, version strings, model parameters — switch to --font-mono so columns of numbers don't wiggle.

③ Eyebrows in 10/UPPER.

Section eyebrows are always --fs-10, 0.18–0.20em letter-spacing, uppercase, accent-pink color.

④ Negative tracking on display.

Anything ≥ 28px gets letter-spacing: -0.015em to -0.04em. Anything below 16px never gets negative tracking — it kills CJK glyph balance.

⑤ Mix scripts mid-line.

Latin + CJK on the same line is fine; the OS stack handles vertical alignment. Don't add `font-feature-settings: "palt"` — it ruins kana spacing.

⑥ Never italic Han.

CSS italic on Chinese/Japanese glyphs is a synthetic slant and looks broken. Use the accent color for emphasis instead.
