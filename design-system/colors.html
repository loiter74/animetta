Colors

Animetta lives on two axes: a deep night-violet surface scale from black to plum, and a single warm sakura-pink accent that does almost all the talking. Anything else is a guest, used in a strict role.

Primary accent
#e879a8
--c-accent
Used for every call-to-action, every glow, every "Anima is paying attention" signal. If a screen has no pink, something is wrong — but if a screen has more than ~10% pink coverage, something is wrong too.

Surface scale

Four steps, each ~10% lighter than the last. Used as a stacking order — bg → surface → panel → card. Anything that floats higher than --c-card should get a border or glow instead of another lighter fill.

Text scale

Three steps — never four. Muted is for timestamps and disabled states; dim is for body and labels; text is reserved for the thing the user is actually reading.

Glows & borders

Animetta's depth comes from glow + border, not shadow. Glows are always the accent tinted to 15–40% alpha and never blurred more than 12px — softer makes the UI feel hazy, harder makes it feel neon.

rgba(255,255,255,0.08)

Where each color goes

Memorize the table or the system collapses into noise. If a new component needs a color outside this list, escalate it — don't invent.

RoleTokenUsed for
App background --c-bg Window backdrop. Almost never directly visible — sits under glass panels and a scene background image.
Surface --c-surface Glass panels (at 70% alpha), main sidebar, modal substrate.
Panel --c-panel Nested panels inside a glass: input field, dropdown surface.
Card --c-card List rows, meme cards, hovered selection.
Accent --c-accent Primary buttons. Send icon. Active tab underline. AI bubble border. Glow source.
Cyber blue --c-blue User chat bubble (at 20% alpha). Never as a fill on its own.
Success --c-success Connection indicator dot. Saved-state toast. Audio "you spoke" pulse.
Warning --c-warning Latency-over-threshold pill. Unsaved-changes badge.
Error --c-error Disconnected. Voice "cannot hear" hint. Live2D failed-to-load message.
